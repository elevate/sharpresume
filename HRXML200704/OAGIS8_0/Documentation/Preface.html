Preface

This document continues the work of the Open Applications Group Specification Teams since they were first formed in the fall of 1995.  It reflects many hundreds of person hours contributed by the Open Applications Group, Inc. member companies. 

 These people represent the persons who are building, testing, and implementing the business applications thousands of companies are using world wide to run their businesses.  Many customers and other members of the global software community have also contributed to this effort.

 Release 8.0 of OAGIS is not the sum total of all of the interoperability points between business applications but it adds some important and exciting new technical enhancements including:

  • The adoption of long tag names for all Attributes, Field, Compounds, Components, Nouns, Business Object Document.

  • Fully support the W3C XML Schema Recommendation Released May 2, 2001 by taking full advantage of the typing system, and extension capability XML Schema provides. 

  • The use of the other core XML technologies including XSL and XPath to provide constraints on the BODs and to provide targeted action on the Noun by a given Verb.

There is a great deal more work to be done and this document will continue to be updated.  Much care has been given to make it as correct as possible.  If you find any errors, or if you have any comments, please do not hesitate to contact us at xml@openapplications.org.. 

Since each chapter of this document is viewable through a browser and provides hyperlink throughout the documentation it is necessary to unzip the entire zip file in a directory and maintain the hierarchy provided in the zip file. Breaking this directory structure will give you unpredictable results. Additional information about OAGIS 8.0 can be obtained from our Web Site at www.openapplications.org/oagis.

 We here at the Open Applications group hope you will find this document useful.
